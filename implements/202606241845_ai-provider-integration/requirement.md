# AI Provider 接入需求

## 目标

在不改动现有核心诊疗链路的前提下，为平台接入可切换的外部大模型 provider，首期支持豆包与通义千问，保留本地规则与本地模拟作为兜底。

## 范围

- 支持按 `AIConfig.provider` 切换 provider。
- 支持分诊、诊疗建议、病历生成、处方审核解释四类 AI 任务。
- 支持 SSE 流式输出沿用现有会话机制。
- 支持调用记录落 `AICallRecord`。
- 支持 provider 不可用时回退到本地规则/本地模拟结果。

## 非目标

- 不改造现有主业务状态机。
- 不直接把 API Key 暴露给前端。
- 不在首期做微服务拆分。

## 验收点

- 配置豆包或通义后，业务层不需要改代码即可切换。
- 外部模型不可用时，系统仍可完成分诊、病历生成、处方审核。
- SSE 仍可正常输出流式内容。
- 管理端可继续查看 AI 调用记录与配置记录。
